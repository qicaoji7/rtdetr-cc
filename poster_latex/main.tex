  % -------------------- ABSTRACT --------------------
  \begin{block}{1. Introduction \& Motivation}
    Urban infrastructure maintenance (e.g., potholes, fallen trees) faces two critical challenges:
    \begin{itemize}
        \item \textbf{Complex Occlusions:} Vehicles or shadows often mimic or hide defects.
        \item \textbf{Long-tail Distribution:} Common defects (cracks) overwhelm rare ones (dead animals).
    \end{itemize}
    
    Current CNN-based solutions (like YOLO) struggle with global context. We propose \textbf{UrbanGuard}, an automated system based on \textbf{RT-DETR (Real-Time Detection Transformer)} \cite{lv2023detr}, achieving an optimal balance between \textbf{high precision (68.7\% mAP)} and \textbf{real-time inference (74 FPS)}.
  \end{block}

  \vspace{1cm}

  % -------------------- METHODOLOGY 1 --------------------
  \begin{block}{2. Data Engineering Pipeline}
    \textbf{A. From Polygon to Bounding Box (MBR)}\
    Raw annotations were provided as segmentation polygons. To adapt them for object detection:
    \begin{itemize}
        \item Developed a pre-processing script to compute the \textbf{Minimum Bounding Rectangle (MBR)}.
        \item Standardized inputs for the RT-DETR encoder.
    \end{itemize}

    \textbf{B. Handling Class Imbalance}\

  % -------------------- ARCHITECTURE --------------------
  \begin{block}{3. Model Architecture: RT-DETR-L}
    Unlike YOLOv8 \cite{jocher2023yolov8}, RT-DETR utilizes a hybrid encoder and an IoU-aware query selection mechanism.
    
    \begin{itemize}
        \item \textbf{Backbone:} HGNetv2 (High performance feature extraction).
        \item \textbf{Encoder:} Efficient Hybrid Encoder (Processing multi-scale features).
        \item \textbf{Decoder:} Transformer Decoder with Auxiliary Prediction Heads.
    \end{itemize}

    \vspace{0.5cm}
